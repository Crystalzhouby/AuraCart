# 优化推荐理由生成

## 现有问题
推荐理由由LLM生成，但是LLM的输入仅有商品的基本信息和用户的提问，而与用户提问相关的商品信息往往是和关于商品的评论、商品信息描述等相关的，但是目前遗漏了这一点。

## 优化策略
目前检索返回了product_id,sku_id和匹配分数，可以将最终检索出来的top-k商品，在语义检索阶段和关键词检索阶段查询到的评论信息也进行返回，作为LLM生成推荐理由的上下文。

# 针对不同来源的商品评论信息做信息优化

## 现有问题
商品评论信息包含商家商品描述，商家FAQ和用户评论，然而这些信息的可信度是不同的，用户评论的真实性和客观性相较商家FAQ等较差，目前的语义检索和关键词检索在处理不同评论信息时应该是平等对待的，需要进行优化

## 优化策略
针对marketing_description、official_faq和user_reviews，其在检索过程中会有一定的得分，最终这些得分会进行累加作为商品的最终分数，为了区分不同来源信息的可信度对最终分数的影响，可以设置不同的权重，最终得分为不同来源评论信息的加权和。
例如fac_marketing_description = 1,fac_official_faq = 1,fac_user_reviews = 0.7。
