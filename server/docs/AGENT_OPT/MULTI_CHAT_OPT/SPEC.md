# 多轮对话中自然语言描述难以变更价格
第一轮对话：推荐300元以下的防晒霜；
第二轮对话：请推荐更平价的产品；
在第两轮对话中，推荐的产品和第一轮一样，只是又加了一个平价的产品进来。
原因：当前在extraction节点提取对话历史中包含的价格数值作为价格意图，当然也考虑了是哪一个价格数值最可能是用户需求，例如用户先说要300的，后说要200的，那么用户意图应该是要200的，现在通过自然语言的价格需求（例如更平价一些）不会实现更改；
改善：大模型不只是机械地提取最可能的数值，会进一步地根据用户用自然语言对这个数值做修改，如果上面第二轮，用户要求请推荐更平价的产品，可以将检测到是最近的数值是300，然后发现下一轮对话“请推荐更平价的产品”，会把max_price适当下调，例如降低1/6，当然下降的幅度需要根据用户需求的强烈程度来决定。
