# 项目说明

## 项目背景&目标
智能导购RAG系统能够理解用户自然语言描述的商品购买意图，并精准匹配用户提出的价格等方面的要求，能够从商品数据库中找到检索到的商品并进行返回，并给出合理的推荐理由。

## 典型用户场景
| 场景类型 | 用户输入示例 | Agent 能力要求 |
| :--- | :--- | :--- |
| 单轮模糊推荐 | "推荐一款适合油皮的洗面奶" | 意图识别 + 属性匹配检索 |
| 条件筛选 | "200 元以下的蓝牙耳机有哪些？" | 结构化参数提取 + 范围过滤 |

# 具体要求

## 商品知识库搭建
| 要求 | 具体说明 |
| :--- | :--- |
| **非结构化数据向量化** | 将商品详情（规格、功能、材质）、广告文案、用户评价转化为可检索的向量索引 |
| **商品信息动态变化** | 商品信息中的描述、价格、库存等可能会发生实时变化，数据库应能够支持实时更新 |
| **数据一致性保障** | 确保 AI 推荐的商品价格、库存等关键参数准确且实时生效 |
| **Chunking 策略设计** | 合理切分商品信息粒度，平衡召回率与精确度 |
商品知识库的待处理的数据可具体参考。\ecommerce_agent_dataset

## 商品检索
| 要求 | 具体说明 |
| :--- | :--- |
| **意图识别** | 区分“随便看看”与“明确购买倾向”，触发不同的检索策略 |
| **RAG 链路可靠性** | 解决大模型“幻觉”问题，严禁 AI 编造不存在的优惠券或功能 |
| **混合检索** | 能够同时支持向量检索（理解用户的购买需求）和关键词检索（精准匹配价格区间、衣服尺码等要求、此外用户可能还有化妆品是日系的、耳机是蓝牙的等类似的需求 |
| **Prompt Engineering** | 设计有效的系统提示词，控制输出格式与风格 |

## 后端工程要求
| 要求 | 具体说明 |
| :--- | :--- |
| **流式 API** | 支持 WebSocket/SSE 长连接，保证对话的低延迟感 |
| **可扩展网关** | 设计向量检索与大模型调用的接口网关，支持后续扩展 |
| **服务稳定性** | 处理并发、超时、降级等工程场景 |

# 初步实现思路

## 技术选型
| 层次 | 可选方案 | 要求 |
| :--- | :--- | :--- |
| **后端框架** | FastAPI | 无 |
| **数据库** | Milvus / Qdrant / Pinecone / Chroma / Weaviate / pgvector / MySQL等 | 能够同时支持向量检索（理解用户的购买需求）和关键词检索（精准匹配价格区间、此外用户可能还有化妆品是日系的、耳机是蓝牙的等类似的需求） |
| **Embedding** | 豆包\千问等平台embedding模型 | 能够良好地表达用户购买需求与商品消息 |
| **大模型** | Doubao-Seed-2.0-lite | 无 |

## 功能链路
1.数据实时更新：商品信息上传，能够实时感知到价格和库存的变化；
2.知识库搭建：如何把商品信息较好地chunk化，支持关键词检索和语义检索；
3.知识库检索：如何提取检索句中的关键词和语义，去知识库中做检索；
4.多轮对话（扩展功能）：主要是如何存储对话过程中产生的商品信息上下文；
