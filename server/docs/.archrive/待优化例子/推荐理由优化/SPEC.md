# 问题

在同目录的example.txt中我给出了当前通过/api/search接口查询的问题以及接口返回的结果，我有以下疑问？

当前商品推荐顺序应该是按RRF综合排名从第一到最后一名来排序的吧（**请确认一下推荐顺序是怎么排的？**），为什么对于问题1.推荐一款200元以下的防晒霜，其推荐理由中的商品只包含了排序中间的一款？对于问题2其推荐理由也有问题，首先它只包含了一个产品，没有对所有推荐商品都做说明，此外，用户提问中包含了不含酒精的要求，但是推荐理由没有回应用户的需求（提供给大模型的用户评论中包含是否含酒精的问题）。

# 优化思路

1.在推荐理由中要提到所有返回的推荐商品，都要说明为什么推荐这个商品，**解决该问题需要修改生成推荐理由的提示词**。
2.推荐理由应回应好用户的每一个查询意图，**为了让LLM更好地回应用户需求，将查询解析得到的子查询也放入到用户提示词中**。
