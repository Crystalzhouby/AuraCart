# 闲聊对话历史
现在的新对话历史只有在retrieve节点检索完成后才进行更新；而不会将判定为chitchat的对话加入到历史中，现在进行完善，chitchat分支生成完回答后，也将对话加入到历史中；

# 对话历史管理
采用滑动窗口+LLM生成摘要的方式管理用户的对话历史。
限制滑动窗口的大小为3000token，当超过3000token时，将窗口内最早的30%对话压缩为用户商品购买意图（如果已经生成过摘要，那么连带旧摘要一起压缩），当历史被检索时，返回摘要+滑动窗口中的历史，摘要是时间最早的对话历史。

# 检索对话历史
首先分析有哪些节点检索了对话历史，这些节点在读取对话历史时，需要根据时间的远近，给予历史中不同的对话不同的关注度，优先注意最近的对话，请完善相关的提示词。
