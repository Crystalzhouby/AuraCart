# RAG Design

## 数据粒度

商品以一条商品一条主文档为基本粒度，文本包含商品名、类目、价格、库存、详情、标签和卖点。后续如果详情过长，再按规格/评论/广告文案拆 chunk，并保留 `product_id` 回连商品主表。

## 召回

第一阶段先用关键词和结构化过滤跑通：

- 类目词：洗面奶、耳机、跑鞋、防晒。
- 预算词：200 以下、500 以内。
- 人群/场景：油皮、通勤、跑步、三亚度假。
- 否定词：不要、排除、除了、不含。

第二阶段接 Chroma：商品文本向量化后写入 `CHROMA_PERSIST_DIR`，召回 top-k 后再做价格、库存、否定条件过滤。

## 反幻觉

- Prompt 要求只基于检索结果回答。
- 商品卡片由后端结构化返回，客户端不解析模型文本。
- 价格、库存、优惠、商品 ID 只从商品库读取。
- 检索为空时返回“不足以推荐”，并主动追问。
